# ⚙️ Configuração de Variáveis de Ambiente - Create React App

## 📍 Problema Identificado

**Create React App** só carrega variáveis de ambiente em **tempo de build**. Se você adicionar uma variável no `.env` durante a execução, ela não será visível até o servidor ser **reiniciado**.

---

## ✅ Solução

### 1️⃣ Verificar se Variáveis Existem no `.env`

Abra `c:\Calango-bot\.env` e procure por:

```env
REACT_APP_FIREBASE_BUCKET=calango-chatbot.firebasestorage.app
FIREBASE_BUCKET_URL=calango-chatbot.firebasestorage.app
SQUAMATA_API_URL=http://localhost:3005
SQUAMATA_API_KEY=seu-token-aqui
```

✅ Se existem, vá para passo 2.
❌ Se não existem, adicione agora.

---

### 2️⃣ REINICIAR o Frontend

**MUITO IMPORTANTE**: Depois de editar `.env`, você PRECISA reiniciar o servidor!

**Se está rodando em um terminal:**

```bash
# 1. Parar o servidor (Ctrl + C)
^C

# 2. Reiniciar
npm start
```

**Ou, se estiver rodando em background:**

```bash
# Procurar processo Node que está rodando o frontend
netstat -ano | findstr :3000  (Windows)
lsof -i :3000  (Linux/Mac)

# Matar o processo
taskkill /PID <PID> /F  (Windows)
kill -9 <PID>  (Linux/Mac)

# Reiniciar
npm start
```

---

### 3️⃣ Verificar se Variáveis São Carregadas

Abra DevTools (F12) → Console e execute:

```javascript
console.log('REACT_APP_FIREBASE_BUCKET:', process.env.REACT_APP_FIREBASE_BUCKET);
console.log('REACT_APP_API_URL:', process.env.REACT_APP_API_URL);
console.log('Todas as variáveis REACT_APP:', 
  Object.keys(process.env).filter(k => k.startsWith('REACT_APP_'))
);
```

✅ Se mostrar o valor correto, está funcionando!
❌ Se mostrar `undefined`, vá para passo 4.

---

### 4️⃣ Se Ainda Não Funcionar

**Opção A: Limpar cache**
```bash
# Deletar node_modules e cache
rm -rf node_modules
npm cache clean --force
npm install
npm start
```

**Opção B: Usar arquivo `.env.local`**

Create React App também lê `.env.local` (que não é versionado):

```bash
# Criar arquivo .env.local
echo REACT_APP_FIREBASE_BUCKET=calango-chatbot.firebasestorage.app >> .env.local
echo REACT_APP_API_URL=http://localhost:3001 >> .env.local

# Depois reiniciar
npm start
```

## 📋 Nomes Corretos das Variáveis

| Onde | Nome | Valor |
|------|------|-------|
| Backend (.env) | `FIREBASE_BUCKET_URL` | `seu-bucket.firebasestorage.app` |
| Frontend (.env) | `REACT_APP_FIREBASE_BUCKET` | `seu-bucket.firebasestorage.app` |
| Backend (.env) | `SQUAMATA_API_URL` | `http://localhost:3005` |
| Backend (.env) | `SQUAMATA_API_KEY` | `seu-token-aqui` |
| Frontend (.env) | `REACT_APP_API_URL` | `http://localhost:3001` |

## 🔍 Procedimento de Debug

Se ainda tiver problema:

### Passo 1: Verificar arquivo .env

```bash
cat .env | grep REACT_APP_FIREBASE
```

Deve mostrar:
```
REACT_APP_FIREBASE_BUCKET=calango-chatbot.firebasestorage.app
```

### Passo 2: Verificar se frontend está recebendo

Console do navegador (F12):
```javascript
console.log('process.env:', process.env);
// Procure por REACT_APP_FIREBASE_BUCKET
```

### Passo 3: Testar upload e ver erro detalhado

1. Tente fazer upload de imagem
2. Abra DevTools → Console (F12)
3. Procure por erro `[uploadHelper]`
4. Copie o erro completo

### Passo 4: Forçar recarregamento do .env

Se nada funcionar:

```bash
# 1. Kill todos os processos Node
taskkill /F /IM node.exe  (Windows)
pkill -f node  (Linux/Mac)

# 2. Deletar cache da porta 3000
netstat -ano | findstr :3000  (Windows)
lsof -i :3000  (Linux/Mac)

# 3. Aguardar 5 segundos
# 4. Reiniciar tudo
npm start
```

## 🚀 Verificação Rápida

Execute isto no console (F12) para validar tudo:

```javascript
// Verificar que todas as variáveis estão carregadas
const required = {
  'REACT_APP_FIREBASE_BUCKET': process.env.REACT_APP_FIREBASE_BUCKET,
  'REACT_APP_API_URL': process.env.REACT_APP_API_URL,
};

Object.entries(required).forEach(([key, value]) => {
  if (value) {
    console.log(`✅ ${key}: ${value}`);
  } else {
    console.error(`❌ ${key}: NÃO CONFIGURADO`);
  }
});
```

## 📝 Resumo

| Problema | Solução |
|----------|---------|
| `REACT_APP_FIREBASE_BUCKET undefined` | Reiniciar frontend (`npm start`) |
| `.env` editado mas não carregou | Kill processo + npm start |
| Erro "não está configurado" | Adicionar variável com prefixo `REACT_APP_` |
| Funciona local mas não em produção | Garantir `.env` está no servidor de produção |

---

**Solução mais rápida: Apenas REINICIE o frontend!** 🔄
